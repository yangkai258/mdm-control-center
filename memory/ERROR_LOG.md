# 错题本 - ZG 教训记录

> 每犯一个错，就写下来。以后不再犯同样的。

---

## 2026-04-06

### 1. Vite Preview 懒加载 Chunk 不稳定 ⚠️
**现象**：点击多个菜单后页面变空白，刷新恢复  
**根因**：`vite preview` 的静态文件服务在快速连续请求懒加载 chunk 时失败  
**教训**：开发阶段永远用 `npm run dev`，preview 只验证生产构建  
**状态**：排查中

---

### 2. 父路由必须有组件 ⚠️⚠️
**现象**：子路由页面空白，控制台无错误  
**根因**：父路由只有 `redirect`，没有 `component`  
**教训**：所有父路由都要指定 `component: ParentLayout`  
**预防**：路由设计时检查每个路由的 component 字段

---

### 3. 视图文件和路由必须同时存在 ⚠️
**现象**：菜单点击后空白  
**根因**：路由指向的 `.vue` 文件不存在  
**教训**：新建路由时同时创建对应的视图文件  
**预防**：建立路由-文件对应检查脚本

---

### 4. keep-alive 缓存导致 Component undefined ⚠️
**现象**：页面导航后有时显示空白  
**根因**：`keep-alive` 缓存的组件再次激活时 `Component` 为 `undefined`  
**教训**：`v-if="Component"` 守卫是必须的  
**预防**：使用 keep-alive 时做好 undefined 兜底

---

### 5. locale 键必须与路由完全匹配 ⚠️
**现象**：菜单显示 `menu.xxx.yyy` 这种 key  
**根因**：locale 文件缺少对应翻译  
**教训**：新增路由时同步添加 locale 键  
**预防**：建立 locale 键和路由的对应检查表

---

### 6. API Stubs 与视图文件同等重要 ⚠️
**现象**：运行时 import 失败，页面崩溃  
**根因**：视图组件 import 了 API 但文件不存在  
**教训**：批量创建视图文件时，同步创建 API stubs  
**预防**：路由和 API stub 必须同时创建

---

### 7. Vite Dev vs Preview 性能差异 ⚠️
**现象**：preview 模式下 OOM 被 kill  
**根因**：preview 懒加载机制不稳定，长时间运行后进程崩溃  
**教训**：开发阶段用 `npm run dev`，preview 只用于最终验证  
**预防**：不要用 preview 做长时间开发测试

---

### 8. Git 浅克隆问题（历史） ⚠️⚠️
**现象**：GitHub 有文件但本地没有  
**根因**：用了 `--depth=1`  
**教训**：永远全量 clone 和 fetch  
**预防**：已记录到 TOOLS.md

---

### 9. Agent 并行导致文件冲突（历史） ⚠️⚠️
**现象**：5个 Agent 并行 push，代码丢失  
**根因**：没有协调机制，多个 Agent 修改同一分支  
**教训**：并行开发必须有文件范围分配 + 串行推送  
**预防**：见 AGENTS.md 的多 Agent 并行规则

---

### 10. 后端构建目录混淆（历史） ⚠️
**现象**：从根目录构建后端，路由不完整  
**根因**：有两个 main.go，根目录和 backend/ 下各有一个  
**教训**：永远从 `backend/` 目录构建  
**预防**：已记录到 MEMORY.md 和 TOOLS.md
